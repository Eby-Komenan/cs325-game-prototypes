Quick fire Simon

  • Originality: For this game, I added a twist to the original electronic game of Simon that requires players to press a button in the right sequence. The twist is the smaller time window and the fact that the game can also ask players to press any button except for the one specified.
  • Technical Merit: Considering that this is my first phaser prototype, the fact that I was able to produce this game at all is the highlight.
  • Prototype Postmortem: I learned the overall behavior of phaser. For lack of time, I was unable to have the game issue commands multiple commands in one session.
  • Prototype Assets: All of the assets were borrowed.
    • Button spritesheet: Button
    • Background Image: Background Image
    • Failure sound: Failure
    • Success sound: Success
  • Prototype Closest Other Game: I borrowed code from this phaser example in order to register button behaviors.
  • High Concept: A one-sentence summary of your game.
  • Theme: None.
  • Mandated Variety: Mouse input, and randomly generated timer.
  • Prototype Goal: Time-based events.
  • Player Experience Goals: Make the player have their reflexes challenged.
  • Gameplay: As the game starts, the player waits for a command that can be issued 3 to 6 seconds after the game starts. The players then has a short window to obey that command (press one of the three buttons).
  • Strategies: Stay focused and wait for the command.
  • Story/Setting/Premise: There is litteraly no story.
  • Target Audience: Players looking to be challenged by a fast-paced game that can throw curveballs.
  • Play Time: 3- 8 seconds.
